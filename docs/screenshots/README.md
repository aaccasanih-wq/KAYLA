# Capturas de pantalla — KAYLA

Carpeta con las capturas para el README y el pitch. Grabar y guardar aquí los siguientes PNGs:

| Archivo | Qué capturar | Cómo |
|---------|--------------|------|
| `dashboard.png` | Dashboard de Streamlit con datos cargados: métricas arriba, recordatorios agrupados por médico, base de pacientes con filtros aplicados y gráfico de distribución por diagnóstico. | Abrir https://kayla2026.streamlit.app → captura completa (⌘ + Shift + 4 o herramienta de captura). |
| `telegram.png` | Mensaje de recordatorio recibido por el médico en Telegram, con formato Markdown (nombre, DNI, diagnóstico, fecha, celular, posta, médico a cargo). | Disparar un recordatorio manual (Actions → Run workflow) o enviar con `backend/test_telegram.py` → capturar el chat del bot. |
| `landing.png` | Landing page: hero + sección de problema + solución + pricing. | Abrir https://aaccasanih-wq.github.io/KAYLA/ → captura de toda la página (⌘ + Shift + 5 → pantalla completa). |
| `sheet.png` | Google Sheet de pacientes con encabezados y filas de ejemplo, y/o el Google Form vinculado. | Abrir el Sheet → capturar encabezados + varias filas. |

## Recomendaciones

- Usar resolución de retina (2x) para que se vean nítidas en el README.
- Ocultar datos sensibles reales: usar pacientes de ejemplo (nombres y DNIs ficticios).
- Tamaño sugerido: ~1600px de ancho.
- Formato: PNG.

Una vez guardadas aquí, el README ya las referencia en la sección "Capturas de pantalla".
